# แผน: กรอง false positive ของทาง tactile ด้วย OAK-D-Lite depth

> สถานะ: **วางแผนไว้ — ยังไม่เขียนโค้ด** (รอทำตอนมี OAK เสียบ)
> ตอนนี้กำลังพัฒนาฝั่ง RGB (color+logic) บน mp4 ก่อน เพราะ **mp4 ไม่มี depth ในไฟล์**

## ปัญหาที่จะแก้
color+logic detector (`scripts/color_path_detect.py`) ใช้งานได้ระดับใกล้ แต่บางครั้งเผลอจับ
ของเหลืองที่ไม่ใช่ทาง (แท็กซี่ / ต้นไม้ดอกเหลือง / ป้าย / คน) → เป็น false positive

## ไอเดียหลัก
ทาง tactile = **นอนราบกับพื้น** ส่วน false positive ส่วนใหญ่ = **วัตถุที่ตั้งขึ้นจากพื้น**
→ ใช้ depth แยก "ราบกับพื้น" ออกจาก "ลอย/ตั้งขึ้น" ได้

## สถาปัตยกรรม
```
RGB ──► color detector (ของเดิม) ──► candidate เหลือง
Depth ─────────────────────────────► ▼
                          ดึง 3D points ของแต่ละ candidate มาเทียบ "ระนาบพื้น"
                                      ▼
                      ราบกับพื้น = เก็บ / ลอยสูง = ตัด
                                      ▼
                   อันที่เหลือ = ทาง → คำนวณ ระยะ(m) + ทิศ
```

## logic หลังได้ position (3D points)
1. **หาระนาบพื้น (ground plane)** — fit ด้วย RANSAC จาก depth ช่วงล่างของภาพ
   (ไม่ต้องกรอกความสูง/มุมกล้อง — หาเองจากภาพ เหมาะกับถือกล้องมุมไม่แน่นอน)
2. **เช็คแต่ละ candidate ว่า ราบ/ตั้ง** — คำนวณความสูงเหนือพื้นของจุด 3D
   - จุดส่วนใหญ่สูง ≈ 0 (±~10 ซม.) → ราบ → **เก็บ** (เป็นทาง)
   - มีจุดสูงเกิน ~15–20 ซม. เยอะ → ตั้งขึ้น → **ตัด**
   - ใช้ค่ามัธยฐาน/เปอร์เซ็นต์ ไม่ใช้จุดเดียว (กัน noise)
3. **อันที่รอด (= ทาง) → คำนวณนำทาง**
   - ระยะข้างหน้า = Z จุดใกล้สุด → "ทางห่าง 1.5 ม."
   - เยื้องซ้าย/ขวา = X ของ centerline → "เยื้องซ้าย 0.2 ม."
   - ทิศ = fit เส้นตรงกับ centerline 3D → มุมเลี้ยว
4. **กัน error** — ถ้า candidate ไม่มีจุด depth พอ (พื้นเรียบ/ไกล depth เป็นรู)
   → ไม่ฟันธงตัด แต่ fallback ไปใช้ logic 2D เดิม (เส้น horizon)

## ค่า default ที่เสนอ (ยังรอคอนเฟิร์ม)
- เริ่มจาก RANSAC fit พื้นอัตโนมัติ (ไม่ต้องกรอกความสูงกล้อง)
- threshold ตัด "ของตั้ง" = สูงเกิน ~15 ซม. (ทำเป็น argument ปรับได้)
- output: วาดบนจอ (เขียว=ทางบนพื้น / แดง=โดนตัดเพราะลอย) + ตัวเลขระยะ/ทิศ มุมจอ

## ข้อจำกัดที่ต้องจำ (พูดตรง ๆ)
- **ใช้ได้เฉพาะกล้องสด OAK** — mp4 ไม่มี depth, เล่น mp4 จะเห็นแค่ส่วน color
- **ของเหลืองที่ราบกับพื้น depth ตัดไม่ได้** (เส้นจราจรเหลือง / สีทาพื้น / กระเบื้องเหลืองอื่น)
  เพราะอยู่ระนาบเดียวกับทาง → ต้องพึ่ง RGB (รูปทรง/ความต่อเนื่อง/ลายเส้น) ต่อ
- depth บนพื้นเรียบ/มุมเฉียด/ไกล จะ noise/เป็นรู → near range (~1–3 ม.) เท่านั้นที่เชื่อได้

## วิธีทดสอบในห้อง (ไม่ต้องมีทางจริง)
ชี้กล้อง OAK ลงพื้น แล้วลอง:
| ทำ | ควรได้ |
|---|---|
| กระดาษ/เทปเหลือง วางราบพื้น | เก็บ ✅ |
| ถือของเหลืองยกขึ้นกลางอากาศ | ตัด ❌ |
| ขวด/กล่องเหลือง วางตั้งบนพื้น | ตัด ❌ |

## เครื่องมือ
- DepthAI (depthai 3.x) — มี `StereoDepth`, และ `SpatialLocationCalculator`/`SpatialDetectionNetwork`
  ที่คืน XYZ (มม.) ของ ROI จาก depth ได้บน device
- รันบนเครื่องที่เสียบ OAK (ไม่ใช่เครื่องเทรนนี้)
- reuse `scripts/color_path_detect.py` (`process()`) เป็นตัวหา candidate ฝั่ง RGB
